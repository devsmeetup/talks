Templating w/ Handlebars

  <img src="avatar.png">
  

</script>

  • Encourages run-time string parsing via .innerHTML

Vanilla Templating





  <img src="avatar.png">

  • We're working directly w/ DOM
  • Resources are still loaded (like the <img>)
  • Style and theming is hard (scope CSS to #mytemplate)
  • Satic, not composed URLs

Tomorrows Templating


<template> contains inert markup!

  • We're working directly w/ DOM
  • Parsed not rendered

var tpl = document.querySelector('#mytemplate');
tpl.content.querySelector('img').src = 'assets/avatar.gif';
document.body.appendChild(tpl.content.cloneNode(true));

.content provides access to the <template>'s ... content

Shdow DOM


  • Provides style and markupd encapsulation
  • Same technology used by browser maker to implement tags like <video> or <select>
element.createShadowRoot() 

<h2>Hello World!</h2>
<script>
  var host = document.querySelector('#sr-one');
  var root = host.createShadowRoot();
  root.innerHTML = '<em>I am inside your div!</em>';
</script>
