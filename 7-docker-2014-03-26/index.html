Was ist Docker?

  • Container: isolierter Prozess
    in 'leichtgewichtiger VM'
  • Dockerfile: beschreibt den Aufbau
    von Containern
  • Images, Docker Index: verteile Container
    mache sie 'überall' lauffähig

Und warum?

  • Server mit vielen Applikationen
    (alle Applikationen Containerizen)
  • Production-Environment zum Testen replizieren
    (baue Production-Container nach jedem Commit neu, auf Entwicklungsmaschine)
  • Teste gegen verschiedene Versionen derselben Library/Datenbank
    (jede Version/Versionskombination = 1 Container)

Warum aber nicht...

...'normale' Virtualisierung (Xen, Virtualbox)?

+ Chef / Puppet / Ansible?

Container vs VM

Container vs VM

Vorteile

  • sehr geringe Startup-Zeit
  • sehr geringer Speicheroverhead
  • native Geschwindigkeit
  • 10-100 Container auf Laptop
  • 100-1000 Container auf Server

(Demo)

Layering

  • Layering von Images mit AUFS
  • Wenn kein AUFS: Alternative Storage Drivers (VFS, BTRFS, ZFS)
  • Images Kompatibel über Storage Drivers hinweg
  • Copy on Write / Layering führt zu sehr kompakten Images
  • Sharing von gemeinsamen Base Images

(Demo)

Docker Index

  • Repository von Images
  • Lassen sich mittels docker pull herunterladen
  • Bzw. als Base Image in Dockerfile verwenden
  • 'Trusted Images'
  • Bsp: paintedfox/postgresql

Beschränkungen

  • Orchestrierung
  • Linking von Containern: sehr beschränkt
  • Umgebung: Kein init-Prozess, kein cron daemon usw.
  • baseimage-docker

boot2docker (Docker on the Mac)

$ brew update
$ brew install boot2docker
$ boot2docker init

# VM hochfahren (Daemon läuft nativ in TinyCoreLinux-Variante)
$ boot2docker up 

# Docker CLI nativ auf dem Mac
$ docker run ubuntu echo hello world
hello world

# Man kann auch per ssh auf die Linux VM
$ boot2docker ssh # (PW tcuser)

Learning (auch über Docker hinaus)

  • 'Funktional': Mache Container immutable
  • Daten außerhalb, als Volume gemounted
  • Container kann jederzeit abgerissen und durch neuen ersetzt werden
  • Führt auch zu mehr Disziplin - alles muss ins Dockerfile!
  • Neubauen vs. Idempotente Operatoren
